과제(3) - ADT

과제 제출 관련 안내 공지사항을 꼭 확인하세요.



1) ADT

- 정수형 분자와 분모로 이루어진 분수 추상 자료형(ADT)을 README.md 파일에 글로 작성한다(이하 "분수").

- 분수는 +, -, *, /의 연산으로 이루어지며, 두 분수의 연산 결과는 하나의 분수로 반환된다. 이때 출력은 기약분수 꼴로 출력한다.

- 또한, 위 기능을 위한 기약분수 연산이 존재하며, 해당 연산은 하나의 분수를 받아 기약분수 꼴로 반환하는 연산이다.

- 추가로 A/B 꼴의 문자열을 입력받아 분수 자료형으로 변환하는 연산, 분수 자료형을 A/B꼴로 출력하는 연산도 작성한다(A, B는 정수).

- README에 작성한 ADT를 바탕으로 my_fraction.h, my_fraction.c 파일을 작성하고, 이를 main.c에 include하여 프로그램을 작성한다.

- 참고: 입력은 띄어쓰기로 구분된 "[분수] [연산자] [분수]" 형태로 입력된다.



입력 2/5 + 2/4

출력 9/10



2) GCD, LCM

- 두 정수를 입력받으면 GCD와 LCM를 구하는 프로그램의 알고리즘 기술 언어, 혹은 Pseudo 코드를 README.md 파일에 글/그림 등으로 작성한다.

- 위 알고리즘을 바탕으로 역시 my_math.h, my_math.c 파일을 작성하고, 이를 main.c에 include하여 프로그램을 작성한다.



입력 18 24

출력 GCD: 6, LCM: 72


3) 직사각형 둘레 및 넓이

- 위 프로그램 라이브러리들을 활용하여, 직사각형의 가로, 세로를 인자로 입력받아 둘레와 넓이를 출력하는 프로그램을 작성한다.

- 해당 프로그램은 scanf 등의 입력 함수 대신, main(int c, char** v)의 구조를 활용하여 외부 인자를 넘겨받아 작동하는 프로그램으로 작성한다(참조: https://github.com/kmu-datastructure-ju/arguments-based-adder). 



입력(CMD) .\program.exe 3/7 4/9

출력 Perimeter: 110/63, Area: 4/21


+ 추가 점수
- 본 과제의 기본 구조는 A/B 형태의 분수 문자열로 자료형 할당 및 출력을 전제하고 있다. 추가로 정수 및 소수 형태로도 숫자를 입력받아 분수 자료를 생성하는 연산을 정의하고, 모든 과제의 입력으로 분수 대신 정수, 소수 형태가 가능하게 코딩해보자.
- 또한, 기약분수 꼴로 변환하였을 때 분모가 1인 경우, 분수 대신 정수가 출력되도록 구조를 업그레이드 해보자.
- 본 과제에서 명시한 조건 내에서 런타임 오류를 일으킬 수 있는 부분이 존재한다. 해당 부분들은 과제 채점 과정에서 입력하게 되며, 오류 핸들링 된 부분당 0.5점의 가산점이 추가로 부여된다.

+ 참고 사항
- 과제의 기본 점수는 세부 과제 한 항목당 5점이며, 추가 점수는 항목당 0.5점의 가산점이 부여됩니다.
- 과제 제출 방법 세부 안내에서 0점 처리로 안내된 조항 및 잘못된/늦은 과제 제출의 정정은 불가합니다.
- 과제 제출 시 과제에 포함된 모든 README.md 파일과 *.h, *.c 파일의 링크를 전부 과제 제출란에 입력하여야 합니다.
