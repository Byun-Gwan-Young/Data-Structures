# 💻 자료 구조 과제: 비트 연산 및 데이터 표현 (Bitwise Operations)

## [과제 내용]

### 과제 1) 시스템 정수 표현 방식 확인
**현재 사용하는 PC 시스템과 라이브러리가 정수를 표현하는 방식이 1의 보수와 2의 보수 중 어떤 것을 사용하는지 알아보는 프로그램을 작성하시오.**
* **핵심 원리:** 비트 반전(`~`) 연산을 통해 시스템이 음수를 처리하는 논리(Two's Complement)를 확인하였습니다.

### 과제 2) 고정 소수점(Fixed-point) 표현 변환
**실수를 입력받아 고정 소수점 표현으로 변환한 비트패턴을 출력하는 프로그램을 작성하시오. 단, 전체 비트는 16비트, 정수부 8비트, 소수부 8비트로 가정한다. Q8.8 fixed-point, two's complement**
* **입력값:** -8.5
* **출력값:** `11110110.10000000`
* **핵심 원리:** 실수에 $2^8(256)$을 곱하여 소수점 아래 8비트를 정수 위치로 이동(Scaling)시킨 후, 각 비트를 마스킹하여 출력하였습니다.

### 과제 3) IEEE 754 표현 방식 출력
**실수를 입력 받아서 IEEE 754 표현 방식으로 부호-1, 지수-8, 가수-23 비트 형태로 출력하는 코드를 작성하시오.**
* **입력값:** 5.75
* **출력값:** `0/1000 0001/0111 0000 0000 0000 0000 000`
* **핵심 원리:** `union`을 사용하여 `float`의 메모리 비트 패턴을 그대로 유지한 채 `unsigned int`로 재해석하여 표준 규격에 맞춰 출력하였습니다.

### 과제 4) 문자열 대소문자 상호 변환
**문자열을 입력받아서 대문자는 소문자로, 소문자는 대문자로 변환하고, 나머지는 그대로 출력하는 코드를 작성하시오. 단, ctype.h 등의 라이브러리는 사용하면 안된다.**
* **입력값:** Hello, Data Stucture!!!
* **출력값:** `hELLO, dATA sTUCTURE!!!`
* **핵심 원리:** 아스키 코드에서 대문자와 소문자의 차이가 32($2^5$)라는 점을 이용, XOR(`^`) 비트 연산을 통해 조건문 분기 없이 5번 비트를 토글하여 변환하였습니다.
